GIT - System kontroli wersji - VCS

"Version Control Systems"



Szkieletem GitHub jest rozproszony system kontroli wersji Git.

Domyślnie na hostingach serwisów internetowych np. GitHub tworzone repozytoria Git są widoczne dla wszystkich. Serwis ten udostępnia darmowe i płatne miejsca na swoje repozytoria. Projektanci korzystających z Git-a decydują się publicznie udostępniać swoją pracę od samego początku tworzenia projektu, aby przyciągnąć uwagę i wcześnie skorzystać z udziału społeczności co do rozbudowy i poprawności działania kodu na różnych urządzeniach i systemach. Niektóre grupy projektantów wolą pracować prywatnie nad projektami, dopóki nie będą gotowe opublikować publicznie swoją pracę. Prywatne repozytoria zapewniają, że praca jest ukryta, i ogranicza współpracę tylko do tych użytkowników, którzy mają dostęp do repozytorium. Te repozytoriom można następnie upublicznić na późniejszym etapie, na przykład po testach na poprawne ich działanie.

W niektórych przypadkach, gdy współpraca miała wyłącznie charakter prywatny, niektóre repozytoria mogą nie zostać nigdy publicznie udostępnione. Korzystanie z GitHub lub podobnego systemu kontroli wersji (śledzenie nie zastępuje dobrego zarządzania projektem); jest to rozszerzenie, usprawnienie dobrego zarządzania projektami i plikami. Jednym z praktycznych aspektów korzystania z GitHub jest radzenie sobie z dużymi plikami np. binarnymi. Pliki binarne, takie jak obrazy, wideo, pliki wykonywalne lub inne nieprzetworzone dane, wykorzystywane w bioinformatyce, są przechowywane w Git jako jeden duży obiekt. W rezultacie każda zmiana, nawet jeśli jest minimalna, prowadzi do kompletnej nowej kopii pliku w repozytorium.

GitHub opiera się na znanym i otwartym systemie kontroli wersji Git, pierwotnie zaprojektowanym przez Linusa Torvaldsa do opracowywania jądra systemu Linux, a teraz rozwijanego i utrzymywanego przez społeczność Git. Jednym z powodów sukcesu GitHub jest to, że oferuje coś więcej niż prostą usługę hostingu kodu źródłowego. Zapewnia programistom i badaczom dynamiczne i oparte na współpracy środowisko, często określane jako platforma do kodowania społecznego, która wspiera recenzowanie, komentowanie i dyskusję. Szeroki wachlarz wysiłków, od pojedynczych do dużych projektów bioinformatycznych, repozytoriów laboratoriów, a także globalna współpraca, sprawiły, że GitHub stał się produktywnym miejscem do dzielenia się kodami i pomysłami oraz do współpracy przy ich tworzeniu.

Szkieletem GitHub jest rozproszony system kontroli wersji Git. Każda zmiana, od naprawienia literówki po całkowite przeprojektowanie oprogramowania, jest śledzona i jednoznacznie identyfikowana. Chociaż Git ma złożony zestaw komend i może być używany w raczej złożonych operacjach, nauka zastosowania podstaw wymaga jedynie kilku nowych koncepcji i poleceń, która zapewni solidną podstawę do efektywnego śledzenia kodu i powiązanych treści dla projektów badawczych.

Git jest systemem kontroli wersji, którego programiści używają do śledzenia, i udostępniania kodu.

GIT działa lokalnie przechowując dane w ukrytym folderze na naszym dysku komputera, które nazywamy repozytorium. Można przechowywać kopię lub utworzyć nowe repozytorium online np. na stronie GitHub lub BitBucket udostępniając je prywatnie lub publicznie, ułatwiając zespołowi programistów pracę na tym samym kodzie projektu nie przeszkadzając sobie wzajemnie.

Po zakończeniu pracy programistów nad projektem kod z repozytorium można połączyć, i wydać wersję ostateczną gotowego projektu.
